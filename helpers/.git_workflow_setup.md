# Git & GitHub CLI Workflow Setup

This document describes the persistent configuration in place to prevent the issues encountered during previous PR creation attempts.

---

## What Was Configured

### 1. Git Configuration (`~/.gitconfig`)
**Prevents:** Truncated output, pager issues

```ini
[core]
    pager = cat          # Always show full output
[pager "diff"]
    = cat               # No paging for diffs
[init]
    defaultBranch = main # Consistent default branch
```

### 2. Bash Configuration (`~/.bashrc`)
**Prevents:** Missing environment variables, repeated configuration

```bash
# Force Git to not use pagers
export GIT_PAGER=cat
export LESS=-RXF

# GitHub token (set this with your actual token)
export GH_TOKEN="ghp_YourTokenHere"

# Helper functions:
# - verify_pr()      : Check if PR exists
# - verify_push()    : Confirm branch pushed to remote
# - setup_git_auth() : Configure authenticated remote URL
```

### 3. Automated PR Script (`create-pr.sh`)
**Prevents:** Manual step skipping, incomplete workflows

A fully automated script that:
- Creates branches
- Stages & commits changes
- Pushes to remote
- Creates PRs with title/body
- Verifies each step with `set -e` (exit on error)
- Shows colored status indicators

---

## How This Ensures Persistence

| Component | Persists Across Sessions? | Why |
|-----------|--------------------------|-----|
| `~/.gitconfig` | ✅ Yes | Loaded automatically by Git |
| `~/.bashrc` | ✅ Yes | Loaded on every new terminal session |
| `create-pr.sh` | ✅ Yes | Executable script in home directory |
| Git safe.directory | ✅ Yes | Prevents "unsafe repository" warnings |

---

## Usage in New Sessions

### Option 1: Use the Script (Recommended)
```bash
source ~/.bashrc  # Load environment variables

# Create a PR with all steps automated:
./create-pr.sh "feature-name" "Commit message" "PR Title" "PR Body"
```

### Option 2: Manual Commands (with config already applied)
```bash
# These commands now work reliably due to ~/.gitconfig and ~/.bashrc:
git checkout -b my-branch
git add -A
git commit -m "My commit"
git push -u origin my-branch
gh pr create --title "Title" --body "Body"
```

---

## Required One-Time Action

**Set your GitHub token in `~/.bashrc`:**

1. Edit the file: `nano ~/.bashrc`
2. Find the line: `export GH_TOKEN="ghp_YourTokenHere"`
3. Replace with your actual token: `export GH_TOKEN="ghp_RealToken123..."`
4. Save and reload: `source ~/.bashrc`

---

## Verification Commands

Run these to confirm everything is working:

```bash
# 1. Check Git pager is set
git config --global core.pager

# 2. Check environment variables
echo $GIT_PAGER
echo $GH_TOKEN

# 3. Test the script exists and is executable
ls -la ~/create-pr.sh

# 4. Test GitHub CLI authentication
gh auth status
```

## Summary

| Problem | Solution | Persistence |
|---------|----------|-------------|
| Truncated output | `GIT_PAGER=cat` | `~/.bashrc` + `~/.gitconfig` |
| Missing GH_TOKEN | `export GH_TOKEN=...` | `~/.bashrc` |
| Manual step errors | `create-pr.sh` script | Home directory |
| Unsafe repository warnings | `safe.directory` config | `~/.gitconfig` |

All configurations are now persistent and will apply automatically in every new terminal session.
